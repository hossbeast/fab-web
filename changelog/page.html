v0.5.1.0 - Sat Feb 28 16:59:20 2014 -0700

  • issue 56 : hash varexprs, discard discovery and fabfiles only upon changes
  • issue 57 : fab must write the buildscript, not fabd (file access permissions)
  • issue 58 : successive buildplan stages should see filesystem changes effected by previous stage
  • fix issue with gcc-dep where non-header-file discovered dependencies are reported incorrectly
  • add @install.dist for package maintainers
  • fix issue where $^starttime is 0 when fabfiles are parsed
  • liblistwise should log backtrace when lwopdir is not present (corrupted install)
  • fix a linker error about mismatched symbol alignment on certain builds
  • create symlinks in ipc-dir instead of fab-dir so that user-facing paths are invariant across invocations
  • fix an issue where NXFILE is cleared before file is seen again, causing nonexistent files to not be detected
  • fix crash in liblistwise due to certain operators improperly handling LWX_WINDOWS_ALL (hl, t)
  • logger now writes to stderr instead of stdout
  • new operator, ln
  • fix an issue with --o<N> help output for N > 6
  • add unit tests for commonly-used s/// transform
  • add MINTOKEN and MAXTOKEN to bison/tokens.fab
  • fab should exit with nonzero status when fabd experiences dependency discovery failure
