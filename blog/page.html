Feb 7 2015

I have just tagged the fab-v0.5.0.0 release.

This is the biggest release since I started this project over 2 years ago, both in terms of new features, and code reorganization.

There are two big new features for users. The first is an overhaul of the listwise syntax, which is an important subset of the fabfile syntax. It is now easier to specify input strings that contain special/punctuation characters by allowing for different "scanmodes". There is a new windowing API for listwise transforms, over 30 new operators, and improvements to the ability to use the listwise executable as an interpreter script.

The second big feature is the fabd caching daemon. I have found that the process of parsing the build description and constructing the project dependency graph is expensive, even when discovery results are cached to disk. Previously, that work had to be done every time you invoke fab to do a build. With the new caching daemon, pretty much all of that work is isolated to the initial invocation. This means that incremental builds are really fast.

See the full changelist.

Take a look at what I have planned for v0.6 and v0.7 over at the github project on the way to v1.0.

-Todd
