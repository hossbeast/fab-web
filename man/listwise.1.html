NAME

listwise - list processing utility  

SYNOPSIS

listwise option logexpr transform-string

listwise --help : options summary
listwise --version : version information
listwise --logs : summary of logging options
listwise --ops : listwise(1) operator listing  

DESCRIPTION

listwise takes a (possibly empty) input list and a listwise(3) transform-string as inputs.

listwise executes the specified transformation and outputs the resulting list.

By default, the input list is read from stdin, and the transform-string is the concatenation of all command-line arguments that are not recognized as either options or logexprs.

listwise can be used as an interpreter for #! scripts (see Examples).

 

OPTIONS

 

General

-s
output only selected entries (default)
-a
output entire list, not just selected entries
-k
output entire stack, not just top list
-n
number output rows 0 .. n (default)
-N
number output rows by list index
-Z
separate output rows by null byte instead of by newline
-A
process subsequent path arguments line-wise (ascii) (default)
-0
process subsequent path arguments null-wise
+int
append subsequent non-options with priority int (default : 0)
-t arg
add arg to transform-string (transform)
+t
treat subsequent non-options as -t arg (default)
-T path
read transform-string from path
+T
treat subsequent non-options as -T path
-l arg
add arg to initial list (list)
+l
treat subsequent non-options as -l arg
-L path
read initial list from path
+L
treat subsequent non-options as -L path
-H path
read transform-string and initial list from path (hybrid)
+H
treat subsequent non-options as -H path
 

LOGEXPR

Log expressions control which log messages are enabled

 +logcat
enable log messages tagged with the category logcat

 -logcat
disable log messages tagged with the category logcat
 

Categories

PARSE
display the parsed transform-string
EXEC
step-by-step execution
OPINFO
operator informational messages
 

EXIT STATUS

An exit status of 0 means success.

An exit status > 127 means an error was encountered that is not specific to listwise.

An exit status < 128 means that one of the following listwise-specific errors was encountered.

(1)BADFILE
unable to process input file
(2)BADARGS
invalid command-line arguments
 

EXAMPLES

An example lw script to produce file listings, like ls(1)
